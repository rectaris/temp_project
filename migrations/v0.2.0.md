# v0.2.0 Migration Notes

This release makes Copier the primary long-term template interface.

## Downstream Actions

1. Ensure the generated repository is clean with `git status --short`.
2. Run `copier update`.
3. Review `.copier-answers.yml` changes.
4. Review and resolve any `*.rej` files.
5. Run the generated repository validation commands.
6. Commit the update as one coherent workflow-template refresh.

## Notes

The legacy copy-only template directory was replaced by `template/`. Existing repositories should prefer `copier update` once they have a committed `.copier-answers.yml`.
